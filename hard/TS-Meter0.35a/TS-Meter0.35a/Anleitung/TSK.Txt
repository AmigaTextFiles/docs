English:
========

Full-Doc in German only, sorry! No time for translation.

Usage: TSK <file>

Where <file> is the name (and path) of the .TSD-file (version 1.0) that contains
data of a speaker measured by TS-Meter. A file with the same name and the ending
.Txt that contains the TS-parameters will be written. The graphics are shown
only on screen in this version. Use a screensaver, if you need them in a file.


ATTENTION! This early version maybe contain many bugs and could be dangerous!
           Use at your own risk!!

Deutsch:
========

VORWORT, EINFÜHRUNG
===================

Was ist das?
------------
TS-Konverter (kurz TSK) dient dazu, aus einem mit dem TS-Meter erzeugten
Datenfile die TS-Parameter zu errechnen und außerdem die Impedanzkurven grafisch
darzustellen. TSK verhält sich also ähnlich wie TS-Meter selbst, wenn es ein
.TSD-File wieder einlädt.

Warum ist es notwendig?
-----------------------
Der für TS-Meter (und TSK) verwendete Compiler erzeugt ziemlich inkompatiblen
Code, wenn ein GUI in Verwendung ist. Dieser Code läuft nicht oder nur sehr
schlecht auf CPUs >= 68040. Deshalb ist dieses Shell-Tool nötig, um mit einem
derartigen Rechner an die im .TSD-File steckenden Daten zu gelangen. Das gilt
(zum Teil) auch für emulierte Systeme wie Amithlon. Außerdem ist es unter
Umständen problematisch, TS-Meter selbst auf einem Rechner zu starten, an den
die TS-Meter-Hardware (zur Zeit) nicht angeschlossen ist. Auch hier kann TSK
helfen.

Man kann dank TSK also auf dem "kleinen" Amiga messen und in .TSD-Files
speichern und mittels TSK auf einem "großen" Rechner die Daten auswerten und
weiterverarbeiten. Oder eben bei gerade nicht angeschlossener Hardware auf
bereits vorliegende Meßdaten zugreifen.

Weitere Informationen können evtl. der Anleitung zum TS-Meter entnommen
werden.

SYSTEMANFORDERUNGEN
===================

Der TSK sollte eigentlich auf jedem Amiga ab KS 1.2 mit WB 1.2 laufen, getestet
wurde er bisher aber nur auf folgenden Systemen:

- Amiga 1200,  KS3.1 / WB 3.1, 68030/42,  10MB RAM (2M Chip, 8M Fast)
- Amiga 2000,  KS3.1 / WB 3.1, 68060/50,  66MB RAM (2M Chip, 64M Fast)

Auf folgendem System kam es jedoch zum Totalabsturz (allerdings nicht bei jedem
Aufruf, manchmal arbeitete das Programm normal), warum konnte ich in der Kürze
der Zeit nicht herausfinden:

- Amithlon, KS3.1 / WB 3.9, Athlon XP2400+ (emulierter 68040), 1GB RAM

Wiederum funktionsfähig war er dagegen auf einem UAE 0.8.6 mit KS3.1/WB3.1,
welcher auf obigem Amithlon-System lief.


INSTALLATION DER SOFTWARE
=========================

TSK irgendwo in den Suchpfad kopieren, z.B. nach C:. Außerdem muß der
Systembefehl "Echo" verfügbar sein (resident oder als Datei, je nach
OS-Version).


PROGRAMMSTART
=============

In einer Shell "TSK <file>" eingeben, wobei <file> ein TS-Meter Datenfile 1.0
(üblicherweise mit Endung .TSD) sein muß (z.B. von TS-Meter 0.35 generiert). Es
wird eine Datei mit gleichem Namen aber Endung .Txt geschrieben, die die
TS-Parameter und weitere Informatioen als Text enthält. Die Grafik (die
Impedanzkurven) werden in dieser Version nur auf dem Bildschirm dargestellt.
Sollen sie weiterverwendet werden, so ist ein Screensaver zu bemühen.

TSK benötigt das Init-File (Parameter-File) S:TSK-Init, welches, sofern es
fehlt, mit Defaultinhalt erzeugt wird. Hinweise dazu stehen in diesem File
selbst. Das erzeugte Default-TSK-Init ist u.a. für Amithlonsysteme geeignet.
Für andere Systeme muß es evtl. angepaßt werden.
