Si vais a coger el fichero por file-request, estas son las nuevas normas aplicadas al archivo:

El nombre genérico ahora es: agbbs??html.lha aunque en BBS's que no soporten nombres largos el fichero se llamará agbbs??h.lha

Donde las dos interrogaciones: "??" representan las últimas dos cifras del año.

He decidido emplear las normativas de Aminet con respecto al nombre del archivo. Estas normativas son relativas a sobreescribir la versión más antigua con el mismo nombre, de tal forma que siempre este la última revisión. Así en un mismo CD de Aminet (puesto que estos salen cada dos o tres meses) no tendrá varias revisiones de AGBBS con diferentes nombres.

Para saber cual es la revisión (a que fecha pertenece) es aconsejable dirigirse a la descripción del archivo. En Aminet se usan los archivos con la extensión readme (en el caso de este fichero sería: agbbs??html.readme), en las BBS's se suele usar un archivo llamado FILE_ID.DIZ que automáticamente incluye la descripción en la base de ficheros de la BBS (esto ocurre si el Sysop cuenta con estas características). En estos ficheros se daran los detalles de la fecha de emisión y otros aspectos importantes.
