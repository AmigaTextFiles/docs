No-Frames-Bereich definieren


Beschreibung: so wird's gemacht

An dieser Stelle müssen Sie sich vorstellen, daß es sich bei dem, was Sie hier sehen, um das handelt, was angezeigt wird, wenn Ihr Browser keine Frames interpretiert. Also der <body>-Bereich unterhalb eines Frame-Sets. Anhand der folgenden beiden Verweise können Sie testen, wie ein frame-fähiger Browser mit No-Frames-Bereichen klarkommt.

 tciac.htm
Das ist eine Datei, die eigentlich zu einem Frame-Set gehört. Wenn Sie auf den Verweis klicken, wird die Datei jedoch ohne das Frame-Set angezeigt. Die Datei enthält einen No-Frames-Bereich (gut sichtbar grau hinterlegt). Ihr Browser sollte die Daten innerhalb dieses No-Frames-Bereichs auf jeden Fall anzeigen. Tut er das nicht, interpretiert er vermutlich Frames, versteht das <noframes>-Tag aber wohl falsch.

 tciaa.htm
Das ist eine Datei, die ein Frame-Set definiert. Innerhalb des Frame-Sets wird rechts im Hauptfenster die Datei tciac.htm angezeigt, also die Datei mit dem No-Frames-Bereich. Wenn Ihr Browser nicht nur Frames interpretiert, sondern auch No-Frames-Bereiche, sollte er die entsprechenden Daten nicht anzeigen.

Klicken Sie auf beide Verweise, um den Unterschied zu überprüfen. Mit der Zurückfunktion Ihres Browsers kommen Sie wieder hierhin.
