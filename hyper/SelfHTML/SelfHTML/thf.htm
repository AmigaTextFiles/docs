Allgemeines zu downloadbaren Schriftarten

Downloadbare Schriftarten werden nicht im Sinne des jeweiligen Betriebssystems beim Anwender installiert, sondern direkt in WWW-Seiten als Referenz auf eine Schriftartendatei mit eingebunden. Jede Browser-Software, die downloadbare Schriftarten interpretiert, erkennt solche Schriftarten und stellt sie am Anwenderbildschirm perfekt dar. Dazu wird die Schriftartendatei zusammen mit der HTML-Datei auf den Anwenderrechner übertragen. Sie wird jedoch nach derzeitigem Stand nicht auf dem Anwenderrechner installiert, sondern bleibt nur für die Anzeige der betroffenen WWW-Seiten im Arbeitsspeicher des Anwenders. Der Browser kann die Schriftartendaten interpretieren und stellt Texte, die in HTML mit dieser Schriftart formatiert sind, entsprechend dar. Voraussetzung dafür ist natürlich eine grafische Benutzeroberfläche.

Wie bei den meisten neueren Entwicklungen zum WWW-Publishing, so gibt es auch im Fall der downloadbaren Schriftarten zwei inkompatible Lösungen: die eine stammt von Netscape, die andere von Microsoft. Für beide Lösungen gibt es je eine spezielle Software, die es HTML-Autoren erlaubt, Schriftarten in dem Dateiformat zu erstellen, die der jeweilige Browser unterstützt. Beide Programme erlauben es auch, vorhandene herkömmliche Schriftarten wie TrueType- oder Adobe-Schriftarten in das neue Dateiformat zu konvertieren. Um für beide Browser zu schreiben, brauchen Sie beide Programme und müssen für eine gewünschte Schriftart beide neuen Schritartendateien erzeugen. In HTML-Dateien können Sie dann beide Schriftarten einbinden.

 

Netscape und Bitstream/TrueDoc

 Bitstream ist ein Schriftartenkonzept für plattformübergreifendes elektronisches Publizieren und damit auch für den Einsatz im World Wide Web interessant. Bitstream hat zu diesem Zweck eine spezielle Technologie geschaffen: das TrueDoc-Modell. TrueDoc ist ein Entwicklerwerkzeug zum Erzeugen plattformunabhängiger Schriftarten. Solche Schriftarten werden nicht im Sinne des jeweiligen Betriebssystems beim Anwender installiert, sondern direkt in elektronische Dokumente mit eingebunden. Jede Betrachter-Software, die TrueDoc interpretiert, erkennt solche Schriftarten und stellt sie am Anwenderbildschirm dar.

Im WWW sind zum Funktionieren dieser Technologie zwei Voraussetzungen erforderlich:

  1. Der WWW-Browser muß die TrueDoc-Technologie beherrschen,
  2. Die Schriftarten nach dem TrueDoc-Modell müssen nach einer bestimmten Konvention gemeinsam mit HTML-Dateien vom WWW-Server zum WWW-Browser des Anwenders gelangen.

Netscape unterstützt ab der Version 4.01 seines Browsers die TrueDoc-Technologie.

Auf den WWW-Seiten von Bitstream finden Sie zu diesem Thema weitere Informationen:
 Netscape und Bitstream TrueDoc
 Bitstream TrueDoc und HTML

Schriftartendateien dieser Art haben die Dateiendung .pfr ("Portable Font Resource"). Um solche Dateien zu erzeugen oder vorhandene Schriftartendateien in solche Schriftartendateien zu konvertieren, brauchen Sie ein entsprechendes Programm, beispielsweise das Programm HexWeb Typograph. Truetype- bzw. Postscriptschriften können Sie mit diesem Programm mit Hilfe eines "Character Shape Recorders (CSR)" in eine "Portable Font Resource" umwandeln.

Sie können das Programm HexWeb Typograph als Demo-Version auf den WWW-Seiten von  HexMac downloaden. Das Programm ist für Macintosh und Windows 95 verfügbar.

Das Erzeugen der downloadbaren Schriftart wird bei dieser Technologie als "Brennen" bezeichnet. Das Brennen ist nämlich nicht nur ein reiner Konvertiervorgang zwischen zwei Dateiformaten. Beim Brennen müssen Sie auch die URL-Adresse angeben, unter der Sie die Schriftartendatei zur Verfügung stellen wollen. Die Schriftartendatei funktioniert dann nur auf der angegebenen URL-Adresse. Das Klauen von Schriftarten soll dadurch verhindert werden.

Bei Fragen zum Programm HexWeb Typograph wenden Sie sich bitte an den Hersteller, nicht an den Autor dieses Dokuments.

 

Microsoft und das Web Embedding Fonts Tool WEFT

Als Technologie für downloadbare Schriftarten hat Microsoft das Programm WEFT (Web Embedding Fonts Tool) entwickelt. Der MS Internet Explorer unterstützt ab der Version 4.0 seines Browsers diese Technologie.

Sie können das Programm WEFT unterhalb der  Microsoft WEFT Homepage downloaden. An der gleichen Stelle finden Sie auch eine Einführung in die WEFT-Technologie und in die Bedienung des Programms.

WEFT setzt bereits vorhandene WWW-Projekte voraus. Um dieses Programm sinnvoll nutzen zu können, erstellen Sie also zuerst Ihr Projekt und verwenden innerhalb Ihrer HTML-Dateien mit den dafür üblichen HTML- oder CSS-Befehlen gewünschte Schriftartennamen. Dann können Sie den WEFT Wizard starten (Teil von WEFT). Mit diesem Software-Assistenten können Sie das Projekt analysieren lassen. Der WEFT Wizard ermittelt die verwendeten Schriftartennamen. Sie können auswählen, für welche der Schriftarten Sie ein "Font Object", sprich eine Schriftartendatei, erstellen möchten.

Schriftartendateien, die mit WEFT erzeugt werden, haben die Dateiendung .eot. Beim Erzeugen dieser Schriftarten gibt es diverse Optionen, etwa die, ob Sie für jede HTML-Datei eine eigene Schriftart oder eine gemeinsame Schriftart für alle HTML-Dateien wünschen.

Beim Erzeugen der EOT-Dateien müssen Sie auch deren Ablageort angeben. Das kann für WWW-Projekte beispielsweise eine FTP-Adresse sein, auf der Sie Dateien ablegen können. Ferner geben Sie an, welche URL-Adressen diese Schriftartendateien benutzen dürfen. Dies ist eine Maßnahme, die verhindern soll, daß jeder, der will, Ihre Schriftartendateien per Referenz in seinen eigenen Projekten verwendet.

Bei Fragen zum Programm WEFT wenden Sie sich bitte an Microsoft, nicht an den Autor dieses Dokuments.

 

Downloadbare Schriftarten in HTML einbinden

Um innerhalb einer HTML-Datei downloadbare Schriftarten zu verwenden, müssen Sie die Schriftarten mit einem dafür vorgesehenen Befehl referenzieren.

Beispiel 1 - PFR-Dateien nach HTML-Syntax einbinden:

<head>
<link rel=fontdef src="http://www.meine.com/fonts/chianti.pfr">
</head><body>
<font face="Chianti">Text in Beispielschriftart</font>
</body></head></html>

Beispiel 2 - PFR-Dateien nach CSS-Syntax einbinden:

<html><head>
<style type="text/css">
@font-face { font-family:Chianti XBd BT;
             src:url(http://www.meine.com/fonts/chianti.pfr); }
</style>
</head><body>
<p style="font-family:Chianti XBd BT">Text in Beispielschriftart</p>
</body></head></html>

Beispiel 3 - EOT-Dateien nach CSS-Syntax einbinden:

<head>
<style type="text/css">
@font-face { font-family:Chianti XBd BT;  
             src:url(http://www.meine.com/fonts/chianti.eot); }
</style>
</head><body>
<p style="font-family:Chianti XBd BT">Text in Beispielschriftart</p>
</body></head></html>

Erläuterung:

Downloadbare Schriftarten werden im Kopf einer HTML-Datei eingebunden, die solche Schriftarten verwendet.

Eine downloadbare Schriftart nach Bitstream-Technologie (PFR-Datei) können Sie entweder mit einem HTML-Befehl (wie im Beispiel 1) oder mit einer CSS Style-Sheet-Angabe (wie im Beispiel 2) einbinden. Benutzen Sie den HTML-Befehl, wenn Sie die Schriftart mit dem  HTML-Befehl für Schriftarten (<font face=>) arbeiten. Wenn Sie dagegen mit  Schriftarten in CSS Style-Sheets arbeiten, benutzen Sie die Einbindung nach CSS-Syntax. Wenn Sie beide Formen benutzen, binden Sie die Schriftart nach beiden Mustern ein.

Die Style-Sheet-Syntax zum Einbinden von EOT-Dateien nach der Microsoft-Technologie brauchen Sie nicht selbst durchzuführen. Einträge von der Art, wie sie im obigen Beispiel 3 notiert sind, nimmt das Programm WEFT selbständig in den betroffenen Dateien vor. Es ist dann egal, ob Sie die Schriftart innerhalb der HTML-Datei als CSS Style-Sheet-Angabe oder traditionell mit <font face=> verwenden. Der MS Internet Explorer 4 erkennt in jedem Fall den Bezug und überträgt die erforderliche Schriftart mit auf den Rechner des Anwenders.

Beachten Sie:

Allgemeine Beschreibungen zu den Befehle aus den obigen Beispielen finden Sie innerhalb dieser Dokumentation an folgenden Stellen:

 Schriftarten mit Schriftartendatei in CSS Style-Sheets (CSS-Syntax zum Einbinden von Schriftartdateien)
 Logische Beziehungen einer HTML-Datei (HTML-Syntax für logische Verknüpfung zu anderen Dateien)

Die Bitstream Truetype-Schriftarten werden von WWW-Servern nur dann korrekt an den aufrufenden Browser übertragen, wenn in der Konfiguration des WWW-Servers ein entsprechender Mime-Type dafür eingetragen ist. Bitten Sie Ihren Webmaster/Provider/Administrator gegebenenfalls, den Eintrag application/font-tdpfr für Dateien mit der Endung .pfr im WWW-Server vorzunehmen.
