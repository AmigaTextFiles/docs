ARexx-Befehle in Dialogfenstern

REQUESTNOTIFY (PROMPT/S)
Öffnet ein Hinweisfenster, das sich nur durch Anklicken von OK wieder schließen läßt.
  * PROMPT legt den anzuzeigenden Text fest.

REQUESTRESPONSE (PROMPT/K)
Öffnet ein Abfragedialogfenster mit den Tastenfeldern 'Ja' und 'Nein'. Wenn der User sich für 'Nein' entscheidet, erscheint eine Warnmeldung.
  * PROMPT legt den anzuzeigenden Text fest.

ARexx unterstützt eine Variable 'RC', in der von bestimmten Befehlen gelieferte Fehlercodes abgelegt werden. Diese wird normalerweise auf 0 (für 'kein Fehler') gesetzt und liefert nur dann einen positiven Wert, wenn ein Fehler aufgetreten ist. Um zu ermitteln, ob durch das Anklicken von 'Abbruch' ein Fehler gemeldet worden ist, ist wie folgt vorzugehen:

RequestResponse PROMPT 'Weitermachen?'
If (RC > 0) THEN
Exit

REQUESTFILE (TITLE/K,PATH/K,FILE/K,PATTERN/K, DRAWERSONLY/S)

Öffnet ein Dateiauswahlfenster.

Titel, Pfad und Dateiname sowie Suchmuster für die angezeigten Dateien lassen sich individuell festlegen.

Wird der Parameter DRAWERSONLY verwendet, so so werden ausschließlich Verzeichnisse (und keine Dateien) angezeigt.

Pfad und Dateiname werden als ein String in RESULT zurückgegeben. Wenn der Benutzer das Dialogfenster mit 'Abbruch' schließt, so wird eine Warnung ausgegeben.

WIZARDREQ (TITLE/K,LABEL/K,TEXTBOX/K/N id text, CONTENTS/K,BUTTON/K/N id text,BUTTON/K/N id text)

Ermöglicht die Erstellung eines Assistenten innerhalb eines ARexx-Skripts. Diese Art von Dialogfenster ist modal.

Alle Parameter sind optional, ihre Reihenfolge ist beliebig.

TITLE legt den Namen des Dialogfensters fest.

LABEL spezifiziert einen Textstring, der im oberen Rand des Dialogfensters angezeigt wird. Durch bis zu sechs LABEL-Parameter lassen sich bis zu sechs Zeilen Text darstellen.

HINWEIS: Durch Einfügen von 'LABEL ""' läßt sich eine Leerzeile einfügen.

TEXTBOX id text, erstellt ein Texteingabefeld mit einer dazugehörigen Identifikationsnummer und einem Textlabel.

Zur Zeit lassen sich maximal 80 Zeichen eingeben. In einem Dialogfenster lassen sich bis zu acht solcher Textzeilen plazieren.

BUTTON id text, erstellt ein Tastenfeld am unteren Dialogfensterrand mit einer dazugehörigen Identifikationsnummer und einem beliebigen Text. Es lassen sich bis zu fünf solcher Tasten definieren.

Üblicherweise mindestens OK und ABBRUCH. Dem Namen der Taste kann mit einem Unterstrich ein Tastaturkurzbefehl zugewiesen werden. Beispiel: Bei Angabe von "_ABBRUCH" wird die Taste auch durch einen Druck auf die Taste "A" aktiviert.

Dieser Befehl liefert die Identifikationsnummer des angeklickten Tastensymbols, oder -1 bei Anklicken des Schließsymbols.

Beispiel (als eine Zeile anzugeben:)
WIZARDREQ TITLE "My Wizard" LABEL "Mein Assistent" TEXTBOX 1 "_Name:" TEXTBOX 2 "_Addresse:" BUTTON 1 "_Ok" BUTTON "-1" "_Abbruch"

HINWEIS: Bei obigem Beispiel wird dem Abbruch-Symbol ein Wert von -1 zugewiesen. Dies bedeutet, daß beim Anklicken dieses Symbols oder des Schließsymbols ein Wert von -1 zurückgegeben wird. Beide Symbole lassen sich so in einem Zug prüfen.

WIZARD_GETTEXTBOX id (/N)
Dieser Befehl wird nach einem WIZARDREQ-Befehl dazu verwendet, den in eine Texteingabezeile eingetragenen Text zu ermitteln. Geben Sie einfach die ID-Nummer des Feldes an, um den Text in RESULT zu liefern.
