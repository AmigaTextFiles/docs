Instrukcja programu LiGHTWAVE

Tworzenie objektow - MODELER

Tworzenie sceny - LAYOUT

Strona glowna

 

Tworzymy piramidë w nastëpujâcy sposób :

Menu OBJECT
        CONE (naciskamy N)

                parametry:
                        sides=4
                        segments=1
                        axis=y
                        bottom= -500mm
                        top=     500mm
                        center=  0,0,0
                        radii = x,y,z = 500mm

>>  OK  <<  -  nastëpnie  MAKE.   W  ten  sposób uzyskujemy pierwszâ wersjë
piramidy - gotowâ do modyfikacji.

Wybieramy  POINTS  (dól  ekranu)  (points/polygons  -  wybór na czym chcemy
pracowaê) i zaznaczamy punkty podstawy piramidy

Powiëkszamy podstawë piramidy :

Menu MODIFY
        SIZE (naciskamy N)

                parametry:
                        factor=3.5
                        center = x,y,z=0m

>>  OK  <<  -  uzyskujemy piramidë o odpowiednich proporcjach.  Centrujemy
objekt wg.  osi X,Y,Z (CUSTOM, RexxScript/Center.lwm)

Nastepnie nadajemy surface (powierzchnia) objektowi.

Menu OBJECT
        OPTIONS
                Surface=piramida
                Polygons=Automatic, One Side
                reszte pozostawiamy bez zmian.

>> OK <<, Wybieramy POLYGONS (dól ekranu) i zaznaczamy caîy objekt.

Nakîadamy powierzchnie :

Menu POLYGON
        SURFACE
                APPLY

Zapisujemy jako "piramid.lwob"

W   ten   sposób  stworzyliômy  pierwszy  objekt  naszej  przyszlej  sceny.
Oczywiôcie  praca nad tym objektem nie jest skoïczona.  Nastepnâ faza pracy
nad nim bëdzie nadanie mu odpowiednich atrybutów - lecz juû w LAYOUT'cie.

Nastëpnym   objektem   którym  sië  zajmiemy  bëdzie  objekt  podîoûa.
Oczywiôcie  bëdzie  to  jedynie nieuformowane podloûe.  Objekt ten zostanie
uformowany   dopiero   kiedy   nadane  zostanâ  mu  odpowiednie  faktury  w
LAYOUT'cie.   Od  tego objektu obrazków pomocnicznych bëdzie mniej - sâdze,
ûe  podstawowe  funcje  typu  "SIZE"  etc zostaîy w wystarczajâcym zakresie
zobrazowane.

Objekt podîoûa tworzymy w nastëpujâcy sposób:

Menu OBJECTS
        OPTIONS
          Nadajemy nazwe faktury : podloze
                   Nastëpne parametry to :
               objekt ma byê zbudowany z trójkâtów - wybieramy TRIANGLES
               scianki majâ byê jednostronne - wybieramy ONE SIDE
               >>> OK <<<

Menu OBJECTS
        BOX
        po naciônieciu "N" ustawiamy wszystkie parametry Y na 0
        parametry X, Z powinny byê ustawione na standardowe 
        wartoôci czyli :
                  X,Z low= -500mm,    X,Z high= 500mm, segments X,Z= 1
                  >>> OK <<<

Dziëki   temu   uzyskujemy  pîaszczyznë.   Zaznaczamy  wszystkie  poligony,
nastëpnie:

Menu POLYGONS
        - FLIP
        - dziëki temu odwróciliômy VERSORY ku górze. 

Co   to  sâ  VERSORY  ?   Objekt  ktory  tworzymy  skîada  sië  ze  ôcianek
jednostronnych  -  tak  ustaliliômy  na  poczâtku  tworzenia  tego objektu.
Zaznaczone  POLYGONY  ukazujâ  nam  VERSORY które sâ odzwierciedleniem tego
która strona objektu bëdzie widziana.  Przy tworzeniu objektów dwustronnych
operacja  ta nie byîa by konieczna.  Dokîadne wytîumaczenie istoty VERSORÓW
oraz  tego  dlaczego  tam  gdzie  moûna  korzysta  sië  na  ogól ze ôcianek
jednostronnych  zawarte  jest  w dokîadnej intrukcji.  (Instrukcja programu
LiGHTWAVE/MODELER)

Kolejnâ fazâ ksztaîtowania objektu jest jego powiëkszenie:

Menu MODIFY
        - SIZE
          FACTOR=2000

Dziëki czemu uzyskujemy objekt o porzâdanych wymiarach.

Nastëpnie  powielimy  poligony  objektu, tak aby byî zbudowany z 32768
Tak  duûa  iloôê  ôcianek  jest  konieczna  aby  nasze podîoûe miaîo pîynne
przejôcia/byîo w sposób îagodny znieksztaîcone.

Gdy mamy zaznaczone wszystkie poligony objektu :

Menu POLYGON
        - SUBDIV
        - FACED, >>> OK <<<
        - powielamy tak dîugo aû objekt bëdzie liczyî 32768 ôciany.

Nastëpnie  nadajemy objektowi wczeôniej zdefiniowanâ fakturë "podîoûe"
:

Menu POLYGON
        - SURFACE 
        - APPLY 

Wczytujemy  uprzednio  zrobiny  objekt piramidy - zaznaczamy wszystkie
ôcianki  i nastëpnie nadajemy fakturë o nazwie Piramida2 - i zapisujemy pod
nazwâ   "piramid2.lwob".    Po  raz  kolejny  zmieniamy  nazwë  faktury  na
"piramida3"   i   objekt   z   fakturâ   o   tej  nazwie  zap9isujemy  jako
"piramid3.lwob"
