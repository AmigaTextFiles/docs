Comment éviter que les mauvais scripts d'installation ne remplacent le contenu de LIBS: par des versions plus anciennes ?

Cette technique viens de Mathew Hendry, il ne se souvient plus où il l'a trouvé

créez un répertoire nommé NewLibs, comme ceci:

makedir sys:NewLibs

Utilisez un éditeur comme par exemple C:ed pour modifier le fichier s:startup-sequence, et recherchez la ligne suivante:

Assign >NIL: LIBS: SYS:Classes ADD

remplacez la par cette ligne:

Assign >NIL: SYS:NewLibs SYS:Libs SYS:Classes

enregistrez le texte et redémarrez l'ordinateur, après ceci, n'importe quel programme qui installe des libraires dans LIBS:, va les mettre dans le répertoire NewLibs, au lieu du répertoire libs, de cette manière les fichiers libairies présents ne vont pas être remplacés. la version nouvellement installée va être trouvée d'abord. Une fois que vous avez déterminé quelle est la nouvelle version vous pouvez déplacer la librairie dans le répertoire libs en utilisant le Workbench, ou les commandes suivantes:

copy SYS:NewLibs/<librairie> SYS:Libs
delete SYS:NewLibs/<librairie>

Comment connaître la version d'une librairie ?

utilisez la commande suivante:

version <librairie>

ceci vous donnera le numéro de version de la librairie, si vous en avez plusieurs avec le même nom, comme par exemple pour la question précédente,vous obtiendrez le numéro de version de la librairie trouvée en premier.

dans ce cas pour obtenir le numéro de version d'un fichier particulier, utilisez le nom de fichier avec le chemin complet:

version SYS:Libs/<librairie>

ceci vous donnera le numéro de version de la librairie qui se trouve dans le répertoire principal sys:libs.
