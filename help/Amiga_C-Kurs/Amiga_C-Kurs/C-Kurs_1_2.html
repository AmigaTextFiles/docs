Bezeichner, Datentypen und Variablen


Bezeichner sind Namen von Variablen, Funktionen, Makros, Datentypen und so weiter. Es gelten folgende Regeln:

  • Ein Bezeichner besteht aus einer Folge von Buchstaben (A bis Z, a bis z), Ziffern (0 bis 9) und Unterstrichen (_).
  • Das erste Zeichen darf keine Ziffer sein.
  • Groß- und Kleinbuchstaben werden unterschieden
  • Ein Bezeichner darf beliebig lang sein. Signifikant sind in der Regel nur die ersten 31 Zeichen.
  • Schlüsselwörter dürfen nicht als Bezeichner verwendet werden.

// Beispiele:
float Variable_1;   // Erlaubt
 float 1_Variable;   // Verboten
 int auto;           // Verboten
 int mein_auto;      // Erlaubt

Ganz wichtig ist, daß in der Programmiersprache C zwischen Groß- und Kleinschreibung unterschieden wird. Es kann leicht passieren, daß man sich vertippt und dadurch ein Fehler auftritt.

// Beispiel:
int x;
int X;

x = 2;
X = x*3;

printf("%d",x);  // Ausgabe: 2
 printf("%d",X);  // Ausgabe: 6

Variablen dürfen Sie sich wie einen Container mit Beschriftung vorstellen. In C wird eine Variable so deklariert, daß zuerst der Datentyp, wie z.B. int (für Ganzzahlen) oder float (für Fließkommazahlen) und anschließend der Name der Variable (= Bezeichner) geschrieben wird. Es können auch mehrere Variablen vom gleichen Datentyp in einer Zeile, durch Komma getrennt angegeben werden. Ebenso darf man eine oder mehre Variablen sofort bei der Deklaration initialisieren (= einen Anfangswert zuweisen).

// Beispiele:
int x;      // eine Integer- (Ganzzahl) Variable mit dem Namen x
 float y,z;  // zwei Float- (Fließkomma) Variablen mit Namen y und z
 int a=10;   // eine Integer-Variable Namens a, die den Wert 10 erhält

Auch bei der Bezeichnung von Funktionen müßen Sie darauf achten, daß Sie keine bereits belegten Namen verwenden. Das sogenannte "Überladen von Funktionen" ist in der Programmiersprache C nicht erlaubt.

Im Zusammenhang mit externen Namen (Bezeichner von Funktionen oder Variablen mit externer Bindung) kann eventuell es zu weiteren Namensgleichheiten kommen. Wenn Sie die Farbmarkierung des StormC Editors (bzw. eines anderen Editors wie z.B. GoldED) aktiviert haben, werden sowohl die Schlüsselwörter von C, als auch Bezeichner der AmigaOS API farbig hervorgehoben. Auf diese Weise fällt es Ihnen schon beim eingeben des Quelltextes auf, wenn Sie zufällig einen Bezeichner für eine Funktion oder Variable eingeben, der ein Schlüsselwort ist. In einem solchen Fall wird der Bezeichner dann farbig markiert, wie wir bereits bei unserem Anfangsbeispiel (HelloWorld) gesehen haben.

Schlüsselwörter in C
auto enum restrict unsigned
break extern return void
case float short volatile
char for signed while
const goto sizeof _Bool
continue if static _Complex
default inline struct _Imaginary
do int switch
double long typedef
else register union

Weitere Schlüsselwörter sind in den einzelnen Header-Dateien definiert. So ist z.B. printf in stdio.h deniniert.

Wenn Sie im StormC Editor eines dieser Schlüsselwörter anklicken und anschließend die Help-Taste auf Ihrer Tastatur drücken, erscheint die online-Hilfe von StormC in Form eines Amiga-Guide Hypertextes. Dort werden viele Schlüsselwörter der Sprache C und der AmigaOS-API kurz erklärt. Nehmen Sie z.B. unser HelloWorld Beispiel und klicken Sie dort auf das Wort printf. Drücken Sie dann die Help-Taste. Es erscheint eine kurze Beschreibung der Funktion printf.


Klicken Sie sich ruhig ein wenig durch diese online-Hilfe, um sich ein wenig mit der Sprache C und der AmigaOS API vertraut zu machen.
