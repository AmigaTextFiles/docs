Modularisierung


Die Programmiersprache C unterstützt, wie die meisten modernen Programmiersprachen auch die sogenannte Modularisierung. Das bedeutet, wir können Programmteile in separate Datein auslagern.

Wie man Programme in Module aufteilt, demonstrieren wir anhand unseres kleinen Programms zur Berechnung der Potenzfunktion, das wir im Abschnitt über Funktionen gesehen haben.

Zunächst erzeugen wir eine Datei, welche die konkreten Funktionen enthält: Das sogenannte Implementierungsmodul. Diese Datei speichern wir unter dem Dateinamen "Potenzen.c" ab.

/*  Potenzen.c
 *  Funktionen zur Berechnung von Potenzen
 *  Implementierungsmodul
 */
/*  Funktion zur Berechnung der Potenz
 *  Parameter: basis     (Ganzzahl)
 *             exponent  (Ganzzahl)
 *  Rückgabe:  basis hoch exponent (Ganzzahl)
 *
 *  Beispiel:  2 hoch 3 ergibt 8
 *
 */
 int potenz(int basis, int exponent)
{
  int i,ergebnis;

  ergebnis = 1;

  for (i=0;i<exponent;i++)
  {
    ergebnis = ergebnis * basis;
  }

  return ergebnis;

}

/*  Funktion zur Berechnung der Potenz
 *  Parameter: basis     (Fließkommazahl)
 *             exponent  (Ganzzahl)
 *  Rückgabe:  basis hoch exponent (Fließkommazahl)
 *
 *  Beispiel:  0.5 hoch 3 ergibt 0.125
 *
 */
 float f_potenz(float basis, int exponent)
{
  int i;
  float ergebnis = 1.0;

  for (i=0;i<exponent;i++)
  {
    ergebnis = ergebnis * basis;
  }

  return ergebnis;

}

Die Funktionsprototypen kommen in eine Datei Namens "Potenzen.h". Das ist das sogenannte Definitionsmodul. In der Programmiersprache C nennen wir solche Definitionsmodule auch Header-Dateien und speichern sie mit der Endung ".h" ab.

/*  Potenzen.h
 *  Funktionen zur Berechnung von Potenzen
 *  Definitionsmodul: Enthält die Funktionsprototypen
 */
//  Funktion zur Berechnung der Potenz
 int potenz(int basis, int exponent);

//  Funktion zur Berechnung der Potenz
 float f_potenz(float basis, int exponent);

Die einzelnen Programmmodule fügen wir mit der Präprozessor-Direktive #include wieder zusammen. Beachten sie auch hier, daß die Definitionen der Funktionen vor Ihrem ersten Aufruf bekannt sein müßen. Deshalb fügen wir unser Definitionsmodul als erstes ein.

Die Präprozessor-Direktive #include wird beim Compilierungsvorgang (im Verborgenen) vom Präprozessor duch den Inhalt der angegebenen Datei ersetzt.

Die Syntax der Präprozessor-Direktive #include lautet: 

#include <dateiname>

oder 
#include "dateiname"

Steht der Dateiname zwischen < und >, wird die angegebene Datei nur in einem festgelegten Verzeichnis gesucht. Dieses Verzeichnis wird von den Einstellungen des Compilers vorgegeben. Diese Variante werden Sie in der Regel nur für Include-Dateien der Standard-Bibliothek von C, wie z.B. stdio.h oder math.h oder Includes von Amiga-Systembibliotheken wie z.B. exec/types.h benutzen.

Bei der Variante von #include, bei der der Dateiname zwischen Anführungszeichen steht, wird zuerst im aktuellen Verzeichnis nach der einzufügenden Datei gesucht. Auch Pfadnamen sind hier erlaubt.

Durch die Modularisierung wird unser eigentliches Programm schön klein und übersichtlich. Ein weiterer Vorteil ist, daß Sie so auf einfache Weise Programmteile in anderen Projekten wiederverwenden können.

/*  Listing 2.3.1
 *  Modularisierung
 *  Demonstration anhand von Potenzfunktionen
 */

#include <stdio.h>

#include "Potenzen.h"
#include "Potenzen.c"

int main(void)
{
  int x,y;
  float a;

  printf("Bitte geben Sie zwei Ganzzahlen ein:\n");
  scanf("%d %d",&x,&y);
  printf("%d hoch %d ist %d\n",x,y,potenz(x,y));

  printf("Geben Sie jetzt zuerst eine Fließkommazahl und dann eine Ganzzahl ein: \n");
  scanf("%f %d",&a,&y);
  printf("%f hoch %d ist %f\n",a,y,f_potenz(a,y));

  return 0;
}

Nachdem Sie die Programmmodule mit der Präprozessor-Direktive #include in Ihr Programm eingebunden haben, können Sie die darin enthaltenen Funktionen wie gewohnt verwenden.
