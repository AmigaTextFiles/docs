Screens


Eine Besonderheit des Amiga Betriebssystems sind die Screens. Neben dem sichtbaren Bildschirm können mehrere, virtuelle Bildschirme - die sogenannten Screens - gleichtzeitig geöffnet sein. Diese liegen dann wie Schichten übereinander und man kann sie durchblättern wie die Seiten eines Buches. Auf einem Screen können mehrere Programme ihre grafischen Ausgaben tätigen. Ebenso kann ein Programm mehrere Screens öffnen. Wenn am Amiga mehrere Monitore angeschlossen sind - z.B. einer am onboard Anschluß des Amiga und einer an einer Grafikkarte - können auch mehrere Screens auf verschiedenen Monitoren dargestellt werden.

Ein Screen kann auch größer sein, wie dessen sichtbarer Bereich. Der verborgene Teil kann dann per Scrolling zum Vorschein gebracht werden.

Der Desktop des Amiga ist die Workbench. Diese wird auch auf einem Screen dargestellt - dem Workbench Screen.

Wir können auch eigene Screens öffnen. So können wir zum Beispiel ganz einfach den Workbench Screen klonen d.h. einen Screen öffnen, der die selbe Auflösung, Farbtiefe usw. wir der Workbench Screen hat.

Indem wir den Workbench Screen klonen, können wir auf Nummer sicher gehen, daß der Bildschirmmodus des neuen Screens auch mit der Grafikhardware des jeweiligen Amigas dargestellt werden kann. Wenn der Benutzer für den Workbench Screen einen Bildschirmmodus seiner Grafikkarte ausgewählt hat, wird der geklonte Workbench Screen ebenfalls über die Grafikkarte ausgegeben und nicht etwa über den eingebauten Amiga Grafikchipsatz.

Damit können wir auch so manchen unangenehmen Nebeneffekt vermeiden, der bei der expliziten Angabe von bestimmten Screenmodi entstehen kann. Stellen Sie sich einmal vor, Sie schreiben ein Programm, welches einen 15kHz Pal Screen öffnet. So mancher Amiga Benutzer mit AGA Rechner hat an seinem Amiga einen herkömmlichen Multiscan Monitor angeschlossen, wie er im PC Bereich verwendet wird. Solche Monitore können aber in der Regel keine 15kHz Modi darstellen. Die Folge: Der Benutzer bekommt kein Bild zu sehen. Dieses Problem werden Sie vermutlich schon kennen, vor allem von älteren Spielen.

Zum Öffnen von Screens stellt AmigaOS ähnlich einfache Funktionen bereit wie zum Öffnen von Fenstern. In unserem ersten Beispiel öffnen wir einen neuen Screen mit der Funktion OpenScreenTags:

/*  Listing 7.5.1
 *  Screen Beispiel 1:
 *  Den Workbench Screen clonen
 *
 */
#include <exec/types.h>
#include <intuition/intuition.h>

#include <proto/intuition.h>
#include <proto/dos.h>
#include <proto/exec.h>

struct IntuitionBase  *IntuitionBase = NULL;
struct Screen         *Bildschirm = NULL;         // Zeiger auf Screen Struktur
int main(void)
{
   // Öffne Intuition Library
    IntuitionBase = (struct IntuitionBase *) OpenLibrary("intuition.library", 37L);

   if (IntuitionBase != NULL)
   {
     /* Öffne neuen Screen */
      Bildschirm = OpenScreenTags(NULL,
                                 SA_LikeWorkbench, TRUE,        // Soll wie WB Screen sein
                                  SA_Title, "Screen Beispiel 1", // Screen Titel
                                  TAG_DONE);

     Delay(250);  // Warte 5 Sekunden
     if (Bildschirm != NULL)   // Falls der Screen geöffnet werden konnte...
      CloseScreen(Bildschirm);  // Schließe Screen
     CloseLibrary((struct Library *) IntuitionBase);  // Schließe Intuition Library
   } // end if
   return 0;
}

Wenn wir ein Fenster auf einem eigenen Screen öffnen möchten, müssen wir der beim Öffnen des Screens einen Zeiger auf die Screen Struktur des gewünschten Screens angeben. Das ist genau der Zeiger, der von der Funktion OpenScreenTags zurückgegeben wird. In Listing 7.5.2 übergeben wir dazu der Funktion OpenWindowTags das Tagitem WA_CustomScreen, Bildschirm.

/*  Listing 7.5.2
 *  Screen Beispiel 2:
 *  Ein Fenster auf dem eigenen Screen
 *
 */
#include <exec/types.h>
#include <intuition/intuition.h>

#include <proto/intuition.h>
#include <proto/dos.h>
#include <proto/exec.h>

struct IntuitionBase  *IntuitionBase = NULL;
struct Screen         *Bildschirm = NULL;         // Zeiger auf Screen Struktur
 struct Window         *Fenster = NULL;

int main(void)
{
   // Öffne Intuition Library
    IntuitionBase = (struct IntuitionBase *) OpenLibrary("intuition.library", 37L);

   if (IntuitionBase != NULL)
   {
     /* Öffne neuen Screen */
      Bildschirm = OpenScreenTags(NULL,
                                 SA_LikeWorkbench, TRUE,        // Soll wie WB Screen sein
                                  SA_Title, "Screen Beispiel 2", // Screen Titel
                                  TAG_DONE);

     if (Bildschirm != NULL)
     {
       /* Das folgende Fenster soll auf unserem eigenen Screen erscheinen */
        Fenster = OpenWindowTags(NULL,
                                WA_Width, 300,
                                WA_Height, 200,
                                WA_DragBar, TRUE,
                                WA_Title, "Ein Fenster",
                                WA_CustomScreen, Bildschirm,     // Auf welchen Screen?
                                 WA_Activate, TRUE,
                                TAG_DONE);
     }

     Delay(250);  // Warte 5 Sekunden
     if (Fenster != NULL) CloseWindow(Fenster);

     if (Bildschirm != NULL)   // Falls der Screen geöffnet werden konnte...
      CloseScreen(Bildschirm);  // Schließe Screen
     CloseLibrary((struct Library *) IntuitionBase);  // Schließe Intuition Library
   } // end if
   return 0;
}
