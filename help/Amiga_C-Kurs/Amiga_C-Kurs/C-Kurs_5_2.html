Globale Variablen


Globale Variablen sind im Gegensatz zu Lokalen Variablen im gesamten Programm, also auch in allen Funktionen gültig.

Listing 5.1 ist eine leichte Abwandlung von Listing 5.1 und verwendet x als Globale Variable:

/*  Listing 5.2
 *  Globale Variablen
 */
#include <stdio.h>

int x;  // Variable x ist global
// Funktionsprototyp
 void print_quadrat(void);

int main(void)
{
  extern int x;  // Redundant
   x=5;

  printf("Der Wert der Variablen x ist %d.\n",x);
  print_quadrat();
  printf("Der Wert der Variablen x ist %d.\n",x);

  return 0;
}

/*  print_quadrat schreibt das Quadrat von x an die
 *  Standardausgabe.
 */
 void print_quadrat(void)
{
  extern int x;  // Redundant
   x=x*x;  // Dieses x ist global!
   printf("Das Quadrat von x ist %d.\n",x);
}

Wenn wir eine Variable, wie hier x außerhalb aller anderen Funktionen deklarieren, so ist sie automatisch Global. Mit extern int x deklarieren wir für jede Funktion unseres Programms (hier: main und print_quadrat) explizit als global. Diese explizite Deklaration ist redundant (kann weggelassen werden), falls die Deklaration der entsprechenden Variablen vor ihrer ersten Benutzung erfolgt, was hier der Fall ist. Sie können also die Deklarationen extern int x; in diesem Beispiel auch weglassen.

Die Ausgabe des obigen Programms sieht also wie folgt aus: 

  Der Wert der Variablen x ist 5.
  Das Quadrat von x ist 25.
  Der Wert der Variablen x ist 25.

Im Gegensatz zu Listing 5.1 wird in Listing 5.2 bei der Funktion print_quadrat kein Parameter übergeben. Statt dessen benutzen wir die globale Variable x, die im gesamten Programm verfügbar ist. Dies hat auch zur Folge, daß diese globale Variable durch die Zuweisung x=x*x; unserer Funktion print_quadrat verändert wird. Deshalb gibt der zweite Aufruf von printf("Der Wert der Variablen x ist %d.\n",x); innerhalb der Funktion main den Wert 25 aus.

Allerdings sollte man mit der Verwendung von Globalen Variablen sehr vorsichtig umgehen. Sonst kann es nämlich passieren, daß man zufällig einen Bezeichner für eine Variable wählt, die schon als global deklariert wurde, ohne daß man das will. Dies kann dann zu fehlern führen, weil eventuell eine andere Funktion den Wert dieser Variablen gleichen Namens überschreibt.
