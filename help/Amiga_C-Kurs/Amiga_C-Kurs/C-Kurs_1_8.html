Schleifen mit while


In diesem Abschnitt schauen wir uns die while-Schleife an.

Der Syntax einer while-Schleife ist:

  while( Ausdruck ) Anweisung

Dabei ist der Ausdruck in den runden Klammern die Laufbedingung. Die while-Schleife ist eine sogenannte "kopfgesteuerte Schleife". Das bedeutet, daß zuerst die Laufbedingung bewertet wird. Ist die Laufbedingung wahr, so wird die Schleife ausgeführt, andernfalls nicht.

/*  Listing 1.9
 *  Schleifen mit while
 *  Vertikal-Text
 */
#include <stdio.h>

int main(void)
{
  char buffer[256];  // Puffer für maximal 255 Zeichen plus '\0'.
   int i=0;           // Variable für den Zähler
  printf("Bitte geben Sie einen Text ein (maximal 255 Zeichen): \n");
  gets(buffer);  // Textstring von der Standardeingabe holen
  // Schleife läuft solange, bis das Zeichen '\0' entdeckt wird.
   while (buffer[i] != '\0')
  {

    /*  i-tes Zeichen aus dem Array buffer plus
     *  Zeilenumbruch ausgeben
     */
    printf("%c\n",buffer[i]);

    i++;  // Zähler um 1 erhöhen
  } // Ende der  while-Schleife
  return 0;
}

In Listing 1.9 wollen wir einen String vertikal ausgeben. Damit der Computer weiß, wo der String im Speicher aufhört, ist das letzte Zeichen "\0". Dieses Zeichen müssen Sie nicht eingeben, sondern es wird automatisch als letztes Zeichen eingefügt. Man spricht hier auch von sogenannten "Null terminated strings". Stellen Sie sich das Zeichen "\0" der Einfachheit halber wie einen Pfropfen vor, der das Hintere Ende des Strings abschließt.

Unsere Zählervariable i initialisieren wir mit dem Wert 0. Bei einem Array (der String ist auch ein Array) hat das erste Element den Index 0.

Unsere while-Schleife hat die Laufbedingung (buffer[i] != '\0'). Die Konstruktion "buffer[i]" bedeutet "das i-te Element vom Array buffer". Der Vergleichsoperator "!=" prüft auf Ungleichheit. Wir überprüfen in unserer Laufbedingung also, ob das i-te Element aus buffer ungleich dem Zeichen "\0" (dem Null-Terminator) ist.

Wenn das aktuelle Element also nicht das Zeichen "\0" ist, ist unsere Laufbedingung wahr und der Schleifenrumpf wird ausgeführt.

Der Schleifenrumpf, d.h. alle Anweisungen, die zu der Schleife gehören, befinden sich wie auch bei der for-Schleife in einem Anweisungsblock.

In jedem Schleifendurchlauf geben wir dann das i-te Element aus dem Array buffer plus einen Zeilenumbruch "\n" aus. Dabei fangen wir mit dem ersten Element (Index 0) an und erhöhen den Index bei jedem Schleifendurchlauf mit i++.

Sobald der Index i auf einem Zeichen "\0" steht, bricht die Schleife ab, weil dann unsere Laufbedingung (buffer[i] !='\0') nicht mehr erfüllt ist.
