Zeiger auf Strukturen


Wenn Sie sich schon einmal einen Amiga-spezifischen C Sourcecode angeschaut haben, werden Sie wahrscheinlich häufig auf Deklarationen wie diese gestoßen sein:

 struct Window *MyWindow;

Wie wir in der Lektion über Zeiger und Adressen gelernt haben, muß man bei der Deklaration eines Zeigers auch den Datentyp angeben, auf den dieser Zeigt. In der Lektion über Strukturen haben wir bereits die Definition der Struktur Window aus der Header-Datei intuition/intuition.h gesehen. Dort wurde Window als eine Struktur mit verschiedenen Komponenten definiert. Also bedeutet die Zeile struct Window *MyWindow;, daß wir eine Zeigervariable Namens MyWindow deklarieren, die auf eine Speicherstelle zeigen soll, in der eine Struktur vom Typ Window, (wie in intuition/intuition.h definiert) abgelegt ist.

Das mag im Moment vielleicht noch verwirrend klingen, aber ich denke, Sie werden das hier vorgestellete Prinzip der Zeiger auf Strukturen anhand des folgenden, kleinen Beispielprogramms verstehen. Listing 4.2 ist eine leicht abgewandelte Version von Listing 3.2 und demonstriert die Verwendung von Zeigern auf Strukturen:

/*  Listing 4.2
 *  Zeiger auf Strukturen
 */
#include <stdio.h>

int main(void)
{
  // Strukturtyp Artikel mit 3 Komponenten
   struct Artikel
  {
    char  Bezeichnung[40];  // Name des Artikels
     int   Anzahl;           // Lagerbestand
     float Preis;            // Preis in Euro
   };

  struct Artikel a1,a2;  // Zwei Strukturen a1 und a2 vom Typ Artikel
  struct Artikel *Zeiger;  // Ein Zeiger auf eine Struktur von Typ Artikel
   int Auswahl;             // Ganzzahlige Variable für die Auswahl
  // Jetzt lassen wir unseren Zeiger auf den ersten Artikel zeigen
   Zeiger = &a1;

  /*  Nun lesen wir die Daten für den ersten Artikel ein
   *  Dabei benutzen wir den Zeiger, der gerade auf die
   *  Struktur des ersten Artikels zeigt.
   */
   printf("Geben Sie die Bezeichnung für den ersten Artikel ein:\n");
  scanf("%s",&Zeiger->Bezeichnung);
  printf("Wieviele %s sind im Lager vorhanden?\n",Zeiger->Bezeichnung);
  scanf("%d",&Zeiger->Anzahl);
  printf("Was soll ein %s kosten?\n",Zeiger->Bezeichnung);
  scanf("%f",&Zeiger->Preis);

  // Jetzt lassen wir unseren Zeiger auf den zweiten Artikel zeigen
   Zeiger = &a2;

  /*  Nun lesen wir die Daten für den zweiten Artikel ein
   *  Zuvor haben wir unseren Zeiger auf die Struktur
   *  des zweiten Artikels zeigen lassen.
   */
   printf("Geben Sie die Bezeichnung für den zweiten Artikel ein:\n");
  scanf("%s",&Zeiger->Bezeichnung);
  printf("Wieviele %s sind im Lager vorhanden?\n",Zeiger->Bezeichnung);
  scanf("%d",&Zeiger->Anzahl);
  printf("Was soll ein %s kosten?\n",Zeiger->Bezeichnung);
  scanf("%f",&Zeiger->Preis);

  // Jetzt fragen wir, welchen Artikel der Benutzer ansehen möchte
   printf("Welchen Artikel möchten Sie ansehen (1 oder 2)?");
  scanf("%d",&Auswahl);

  /*  Die Eingabe entscheidet, auf welche Struktur der Zeiger
   *  zeigen soll.
   */
  switch (Auswahl)
  {
    case 1:
      Zeiger = &a1;  // Zeiger zeigt auf ersten Artikel
       break;

    case 2:
      Zeiger = &a2;  // Zeiger zeigt auf zweiten Artikel
       break;

    default:  // Der Benutzer hat etwas falsches eingegeben
       Zeiger = NULL;  // Zeiger wird Nullzeiger
   }

  /*  Falls die Zeigervariable Zeiger nicht auf NULL zeigt:
   *  Daten ausgeben.
   */
   if (Zeiger != NULL)
  {
    // Daten ausgeben
     printf("Artikelbezeichnung: %s\n",Zeiger->Bezeichnung);
    printf("Anzahl: %d\n",Zeiger->Anzahl);
    printf("Preis: %.2f EUR\n",Zeiger->Preis);
  }
  else  // Ansonsten (Zeiger ist Zeiger auf NULL): Fehlermeldung
   {
    printf("Fehlerhafte Eingabe.\n");
  }

  return 0;

}

Zunächst sieht alles vertraut aus: Wir definieren wie in Listing 3.2 einen Strukturtyp Namens Artikel mit drei Komponenten. Anschließend deklarieren wir zwei solcher Strukturen mit struct Artikel a1,a2;.

Was neu ist, ist die Deklaration einer Zeigervariable auf eine Struktur vom Typ Artikel: struct Artikel *Zeiger;. Dies funktioniert nach dem gelichen Prinzip, wie bereits in der Lektion über Zeiger und Adressen besprochen wurde. Die Zeigervariable Zeiger zeigt auf eine Struktur vom Typ Artikel.

Mit Zeiger = &a1; bzw. Zeiger = &a2; lassen wir unsere Zeigervariable auf die Struktur a1 bzw. a2 zeigen.

Mit dem Operator -> können wir auf die Komponenten der Struktur zugreifen, auf die eine Zeigervariable gerade zeigt. So schreiben wir z.B. mit scanf("%s",&Zeiger->Bezeichnung); den String, den der Benutzer eingibt, an die Adresse der Komponenten Bezeichnung derjenigen Struktur, auf die die Zeigervariable Zeiger gerade zeigt.

Bitte beachten Sie den Unterschied: Während wir auf die Komponenten von Strukturen mit dem Operator . zugreifen, brauchen wir, um an die Komponenten von Strukturen, auf die ein Zeiger gerade zeigt, heranzukommen den Operator ->.

Ein Sonderfall in der Programmiersprache C ist der sogenannte Nullzeiger: Während ein Zeiger für gewöhnlich auf eine Adresse im Speicher zeigt, zeigt der Zullzeiger konstant auf NULL. Schauen Sie sich nochmal unsere switch-Verzweigung an:

  /*  Die Eingabe entscheidet, auf welche Struktur der Zeiger
   *  zeigen soll.
   */
  switch (Auswahl)
  {
    case 1:
      Zeiger = &a1;  // Zeiger zeigt auf ersten Artikel
       break;

    case 2:
      Zeiger = &a2;  // Zeiger zeigt auf zweiten Artikel
       break;

    default:  // Der Benutzer hat etwas falsches eingegeben
       Zeiger = NULL;  // Zeiger wird Nullzeiger
   }

Hier wird entsprechend der Eingabe des Benutzers entschieden, wohin unser Zeiger zeigen soll. Stellen Sie sich nun einmal vor, was passieren würde, wenn der Benutzer nicht 1 oder 2 eingibt, sondern einen Wert, der in unseren case-Marken nicht vorkommt. Wenn wir unseren Zeiger nach der default-Marke nicht auf NULL setzen würden, so würde dieser, wenn der Benutzer z.B. 9 eingeben würde, immernoch auf die Struktur a2 zeigen.

Mit if (Zeiger != NULL) können wir nun abfragen, ob der Zeiger im Moment nicht gerade auf NULL zeigt. Wenn diese Bedingung zutrifft, geben wir mit printf("Artikelbezeichnung: %s\n",Zeiger->Bezeichnung); usw. die Komponenten der Struktur aus, auf die der Zeiger gerade zeigt, ansonsten eine Fehlermeldung.

Nullzeiger werden sehr häufig verwendet, um zu vermeiden, daß eine (nicht initialisierte) Zeigervariable nicht irgendwo hinzeigt. Eine weitere Verwendung von Nullzeigern werden wir im Zusammenhang mit Datenstrukturen wie verketteten Listen und Bäumen kennenlernen. Dort dient der Nullzeiger (ähnlich wie das Zeichen '\0' bei den Strings) als eine Art Pfropfen, der sozusagen das Ende der jeweiligen Datenstruktur markiert.
