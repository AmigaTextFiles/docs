Словарь используемых терминов

 24-битовая графика

Режим вывода изображений, характеризуемый тем, что цвет каждой точки на экране дисплея может принимать по 256 значений раздельно для синей, зеленой и красной компонент, что позволяет иметь одновременно 16 777 216 цветов на экране. Понятие палитры для 24-битовых экранов отсутствует, т.к. все цвета доступны одновременно. Амига непосредственно не показывает 24-битовую графику, хотя режим HAM-8 моделей с AGA-сопроцессорами отличить от 24-битового по качеству практически невозможно. Возможность работы Амиги с 24-битовой графикой (иначе называемой true color) в настоящий момент времени дают только дополнительные графические карты, наилучшей из которых является CyberVision 64.

AGA (то же, что AA)

Сокращение от Advanced Graphics Architecture - улучшенная графическая архитектура. Читается "ЭйДжиЭй". Касается значительно более мощного, чем прежний (ECS) набора сопроцессоров Амиги. AGA-сопроцессоры позволяют в любом случае одновременно показывать на экране 256 любых цветов из палитры более 16 миллионов цветов. Старые модели Амиг в обычном режиме показывали не более 64 цветов из палитры 4096. Специальный режим экрана HAM-8 позволяет выводить одновременно 256000 различных цветов из той же палитры в 16 миллионов цветов (HAM-6 старых Амиг давал "только" 4096). AGA позволяет использовать аппаратные спрайты учетверенной ширины (8 байт по сравнению с двумя на старых Амигах) и поддерживает массу экранных режимов с различными частотами разверток. AGA сопроцессоры устанавливаются только в модели A1200 и A4000.

CLI

Сокращение от Command Line Interface (интерфейс командной строки). Читается "СиЭлАй", а не "Кли", как говорят многие. Альтернатива интерфейсу Workbench. Характеризуется тем, что общение с машиной происходит путем набора команд с клавиатуры в консольных окнах, открытых CLI-процессами. Усовершенствованным вариантом CLI, начиная с версии ОС 1.3, является Shell (оболочка).

GUI

Graphic User Interface - графический интерфейс пользователя. Относится к окнам, иконкам, мышке, меню, риквесторам и прочим визуальным атрибутам рабочей среды.

Kickstart

Читается "Кикстарт". Базовый набор функций ОС. На всех моделях, за исключением A1000, содержится в постоянном запоминающем устройстве (ПЗУ). На A1000 находится на загрузочном диске. Характеризуется двумя номерами версий (см. описание команды VERSION). Объем ПЗУ версий до V36 составляет 256 Кб, начиная с V36 - 512 Кб.

NIL:

Специальное логическое устройство Амиги. При обращении к NIL: как к устройству ввода, всегда будет немедленно получено значение EOF (End Of File - конец файла); любой вывод направленный в устройство NIL: уходит в "никуда".

Блиттер

Один из двух сопроцессоров Амиги, который способен со значительной скоростью копировать "прямоугольные" участки памяти с одновременным комбинированием до трех источников в один приемник.

Бутирование (booting)

Процесс запуска и инициализации компьютера. Различают "холодный" и "горячий" старты. Холодный старт происходит после включения питания, когда система абсолютно пуста и надо заново инициализировать все ее составляющие. Горячий старт происходит после сброса (reset) системы нажатием комбинации клавиш "Левая Амига-Правая Амига-Ctrl". В этом случае часть информации, находящейся в памяти, будет сохранена. Некоторые устройства, эмулирующие гибкие диски - например RAD: или SD: при горячем старте полностью сохраняют все свои данные на псевдодисках в памяти Амиги, и с ними можно продолжать работать. Гарантированное исчезновение любых данных, находящихся в памяти, обеспечивает только отключение питания на срок 18 секунд (что может быть актуально при глухих сбоях системы, когда горячего старта недостаточно для ее нормального функционирования, либо при битве с вирусом).

Ввод/вывод

Совокупность способов и устройств, пользуясь которыми компьютер общается с внешним миром. Система ввода/вывода на Амиге включает в себя различные дисковые накопители, накопители, дисплей, параллельный и последовательный порты подключения различных устройств (например, принтера и модема), а также драйверы устройств (т.е. системные программы, обслуживающие конкретные устройства).

Двойная буферизация

Специальный режим воспроизведения изображений, при котором картинка прорисовывается на скрытом экране, который делается видимым только после полного окончания прорисовки; для этого режима требуется двойной объем памяти под два переключаемых экрана. Режим обеспечивает наилучшее качество воспроизведения анимаций, поскольку глазу всегда предъявляется уже полностью прорисованный очередной кадр. Само переключение экранов происходит только в те моменты, когда электронный луч дисплея находится за кадром.

Двойной щелчок

Способ активации объекта, которому соответствует иконка, находящаяся под указателем мышки. Требуется указать объект, затем два раза подряд быстро нажать на левую кнопку мышки.

Драйверы устройств

Специальные программы, обслуживающие конкретные устройства Амиги. Драйверы являются программами более низкого уровня, нежели хэндлеры и обслуживаются системной библиотекой exec.library. Для обмена информацией используется система портов сообщений (message port), поддерживаемая этой библиотекой. Драйверы могут находиться в каталогах SYS:Devs и SYS:Expansion.

Интерактивный режим

Синоним диалогового режима работы, когда некоторая программа периодически останавливается, ожидая ввода пользователем некоторых данных и/или указаний для дальнейших действий.

Коппер

Один из двух сопроцессоров Амиги, обслуживающий в основном дисплейные функции. Коппер начинает исполнение команд всегда с момента начала очередного кадра и синхронизируется в соответствии с движением электронного луча по экрану дисплея.

Логическое устройство

Программно определяемый аналог как бы существующего физического устройства.

Партиция

Один или несколько логических дисков, на которые делятся имеющиеся в системе реальные (физические) жесткие диски. С каждой партицией можно работать, как если бы она была отдельным винчестерским диском. Некоторые компьютерщики называют партиции "разделами".

Переменные окружения

Различные переменные, которые в основном используются в командных файлах. Различают глобальные и локальные переменные окружения. Глобальные переменные являются по сути обычными файлами устройства ENV: (один из каталогов RAM-диска) и доступны для всех программ/процессов без исключения. Локальные находятся в специально резервированных областях оперативной памяти и доступны только из того процесса, который их создал (а также из процессов, порожденных этим процессом).

Пиксель

Наименьший элемент разложения изображения на экране дисплея (от слов Picture Element).

Полный маршрут

Маршрут, который включает в себя имя устройства (через двоеточие). В противном случае маршрут считает неполным. Так, DH0:Графика является полным маршрутом, а Работа/Графика/Чертежи - неполным.

Риквестор

Окошко системного запроса, например, список файлов, из которого необходимо выбрать один или несколько нужных.

Установки (preferences)

Исполняемые программы, находящиеся в каталоге SYS:Preferences и являющиеся частью ОС. Использование этих программ позволяет пользователю сконфигурировать систему наиболее подходящим для себя образом.

Хэндлеры

Программы обслуживания устройств, с которыми работает системная библиотека dos.library. Уровень хэндлеров выше уровня драйверов. Хэндлеры находятся в каталоге SYS:L. Управление хэндлерами происходит с помощью т.н. пакетов.
